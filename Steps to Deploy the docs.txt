1. Add More Content
Create new Markdown files in docs/ (e.g. usage.md, api.md, faq.md).

Front-matter: Give each file an id and title at the top.

2. Update Sidebar (if manual)
If you switched to a manual sidebar, add your new IDs to sidebars.js in the order you want.

If you’re still using autogenerated, just name files and front-matter correctly—Docusaurus will pick them up.

3. Tweak Site Settings
Open docusaurus.config.js:

siteTitle, tagline, favicon

url and baseUrl (for deployment path)

themeConfig.navbar to add or reorder top-nav links

4. Include Images & Videos
Place all assets under static/img/ (or static/videos/).

Reference them in Markdown with absolute paths (/img/..., <iframe> for YouTube).

5. Preview & QA
Run npm run start → browse http://localhost:3000/

Click through every page, check images, links, sidebar order.

Test mobile viewport by shrinking your browser.

8. Embed in Bubble.io
In Bubble editor, drag an HTML element onto your page.

Paste:

html
Copier
Modifier
<iframe
  src="https://YourUserName.github.io/smartreception-docs/"
  width="100%" height="800px"
  frameborder="0">
</iframe>
Preview your Bubble app → your docs appear inline.

9. Automate & Maintain
Auto-deploy on main via GitHub Actions (optional).

Version docs whenever your template changes.

Announce updates on your site or in-app with a “Docs updated” banner.

Follow these steps in order, and you’ll have a fully featured, versioned documentation site live and embedded in your Bubble template.
